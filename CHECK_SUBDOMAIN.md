# Как проверить возможность создания поддомена voice.icesmoke.store

## Способ 1: Через панель управления хостингом

### ISPmanager:

1. Зайдите в панель управления
2. Найдите раздел **"Домены"** или **"Веб-серверы"**
3. Ищите кнопку **"Создать домен"** или **"Добавить домен"**
4. Попробуйте создать поддомен `voice.icesmoke.store`

**Если есть возможность** → ✅ Можно создать поддомен

### cPanel:

1. Зайдите в **"Subdomains"** или **"Поддомены"**
2. Попробуйте создать `voice.icesmoke.store`

**Если есть раздел** → ✅ Можно создать поддомен

---

## Способ 2: Через веб-консоль

### Проверка DNS зоны

```bash
# Проверка DNS записей для icesmoke.store
dig icesmoke.store
nslookup icesmoke.store

# Проверка, кто управляет DNS
whois icesmoke.store | grep "Name Server"
```

**Если DNS управляется хостингом** → ✅ Скорее всего можно создать поддомен

**Если DNS управляется внешним провайдером** → Нужно добавлять A-запись вручную

### Проверка файлов конфигурации

```bash
# Поиск конфигурации поддоменов
ls -la /etc/nginx/sites-available/ | grep voice
ls -la /etc/apache2/sites-available/ | grep voice

# Проверка виртуальных хостов
grep -r "voice.icesmoke" /etc/nginx/
grep -r "voice.icesmoke" /etc/apache2/
```

---

## Способ 3: Проверка через панель DNS

### Если есть доступ к панели управления DNS:

1. Зайдите в панель управления DNS (на хостинге или у регистратора домена)
2. Проверьте, есть ли возможность добавить **A-запись** для `voice`
3. Если есть → ✅ Можно создать поддомен

### Что нужно для поддомена:

- **A-запись**: `voice.icesmoke.store` → IP сервера
- Или **CNAME**: `voice.icesmoke.store` → `icesmoke.store`

---

## Способ 4: Проверка через файловую систему

### Проверьте структуру www/

```bash
# Что в www/
ls -la ~/www/

# Есть ли уже поддомены?
ls -la ~/www/ | grep -E "mail|ftp|www|api"
```

**Если видите поддомены** → ✅ Скорее всего можно создать новый

---

## Быстрая проверка (одна команда):

```bash
# Проверка возможности создания поддомена
echo "=== Проверка панели управления ===" && \
echo "Проверьте в панели хостинга раздел 'Домены' или 'Subdomains'" && \
echo "" && \
echo "=== Проверка DNS ===" && \
dig voice.icesmoke.store +short 2>/dev/null && echo "✅ Поддомен уже существует" || echo "ℹ️  Поддомен не найден (это нормально)" && \
echo "" && \
echo "=== Проверка конфигурации ===" && \
ls -la ~/www/voice* 2>/dev/null && echo "✅ Папка для поддомена существует" || echo "ℹ️  Папка не найдена"
```

## Что делать дальше:

### ✅ Если можно создать поддомен:

1. Создайте поддомен `voice.icesmoke.store` через панель
2. Укажите папку для поддомена (например, `~/www/voice/`)
3. Загрузите код приложения в эту папку
4. Настройте приложение (см. инструкции по развертыванию)

### ❌ Если нельзя создать поддомен:

Вернитесь к варианту с `/voice/` на основном домене

---

## Преимущества поддомена:

- ✅ Не нужно трогать конфигурацию основного сайта
- ✅ Проще настроить (через панель управления)
- ✅ Изолирован от основного сайта
- ✅ Легче управлять

## Недостатки поддомена:

- ❌ Нужен доступ к панели управления DNS
- ❌ Может быть платным (зависит от хостинга)
